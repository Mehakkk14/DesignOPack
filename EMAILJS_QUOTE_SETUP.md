# EmailJS Setup Guide for Quote Request Notifications

This guide will help you set up EmailJS to send email notifications when customers submit quote requests.

## Step 1: Create EmailJS Account

1. Go to [EmailJS Dashboard](https://dashboard.emailjs.com/)
2. Sign up with your email or use Google/GitHub
3. Verify your email address

## Step 2: Create Email Service

1. Click **"Add New Service"**
2. Choose **"Gmail"** (recommended) or your preferred email provider
3. Connect your **rastogimehak3845@gmail.com** account
4. Copy the **Service ID** (e.g., `service_gmail_designopack`)

## Step 4: Get Public Key

1. Go to **"Account"** ‚Üí **"General"**
2. Copy your **Public Key** (starts with underscore, e.g., `_your_public_key_here`)

## Step 5: Update Environment Variables

Update your `.env` file with the actual values:

```env
# EmailJS Configuration
VITE_EMAILJS_SERVICE_ID=service_gmail_designopack
VITE_EMAILJS_TEMPLATE_ID=template_quote_request
VITE_EMAILJS_PUBLIC_KEY=_your_actual_public_key_here
```

## Step 6: Test Email Sending

1. Start your development server: `npm run dev`
2. Go to your website and submit a test quote request
3. Check if the email is received at `rastogimehak3845@gmail.com`

## Troubleshooting

### If emails aren't sending:
1. **Check Console**: Look for EmailJS errors in browser developer tools
2. **Verify Service**: Make sure Gmail service is properly connected
3. **Template Variables**: Ensure all template variables are correctly named
4. **Quotas**: Check EmailJS dashboard for usage limits (200 emails/month free)

### If emails go to spam:
1. **Gmail Settings**: Add your domain to trusted senders
2. **SPF Records**: Configure DNS records for your domain (if using custom domain)

### Common Errors:
- `403 Forbidden`: Check public key is correct
- `404 Not Found`: Verify service ID and template ID
- `400 Bad Request`: Check template variable names match
